# 대시보드 모듈 규칙

## 1. 대시보드 종류
| 대시보드 | URL | 데이터 범위 |
|----------|-----|-------------|
| **글로벌** | `/dashboard` | 사용자가 참여한 모든 프로젝트 |
| **프로젝트** | `/projects/{identifier}` | 해당 프로젝트만 |

## 2. 글로벌 대시보드 위젯
| 위젯 | 데이터 소스 | 표시 형태 |
|------|-------------|-----------|
| 내 이슈 | 내게 할당된 미완료 이슈 | 테이블 (상태별 그룹핑) |
| 최근 활동 | 참여 프로젝트의 Journal (최근 10건) | 타임라인 피드 |
| 상태별 이슈 수 | 전체 이슈 통계 | 도넛 차트 |
| 기한 초과 이슈 | dueDate < today & !isClosed | 경고 목록 |

## 3. 프로젝트 대시보드 위젯
| 위젯 | 데이터 소스 | 표시 형태 |
|------|-------------|-----------|
| 프로젝트 정보 | Project 엔티티 | 카드 |
| 멤버 목록 | ProjectMembership | 아바타 리스트 |
| 이슈 통계 | 상태별/트래커별 카운트 | 차트 |
| 버전 진행률 | Version + 이슈 완료율 | 프로그레스 바 |
| 최근 활동 | 프로젝트 내 Journal | 타임라인 피드 |

## 4. API 응답 구조
```json
// GET /api/dashboard
{
  "myIssues": {
    "open": [...],
    "inProgress": [...],
    "totalCount": 15
  },
  "recentActivity": [...],
  "issueStats": {
    "byStatus": [{ "statusName": "New", "count": 10 }, ...],
    "byPriority": [{ "priorityName": "High", "count": 5 }, ...]
  },
  "overdueIssues": [...]
}
```

## 5. 프론트엔드 규칙
- 위젯은 MUI `Card` + `Grid2`로 배치
- 차트: 경량 라이브러리 사용 (recharts 또는 MUI 자체 차트)
- 데이터 자동 갱신: TanStack Query `refetchInterval` (60초)
- 로딩 시 각 위젯별 독립 `Skeleton`
